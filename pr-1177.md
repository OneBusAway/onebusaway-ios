**Coverage gap for the new hosting path**
`OBAKit/Sheet/Root/WeatherDetailPopupHost.swift`, `OBAKit/Mapping/MapViewController.swift:434-443`

## Strengths

- `@ObservedObject` (not `@StateObject`) is the right call for a wrapper whose view model is owned by the presenting `MapViewController`. Doc comment even calls this out (`WeatherDetailPopupHost.swift:13-21`).
- Dismissal chain is doing the right thing: SwiftUI runs its own scale/opacity exit, then `@Environment(\.dismiss)` tears down the (now-transparent) hosting controller. No `WeakBox`, no closures reaching sideways into UIKit, no double-animation.
- The presentation config (`.overFullScreen`, `.crossDissolve`, `host.view.backgroundColor = .clear`) correctly cedes the visual to `WeatherDetailPopup`'s own dim overlay — no double-dim, and the map stays visible underneath.
- `MapViewController.showWeather()` kept its `@objc private` signature and the `weatherDisplay != nil` guard against the file-scoped mirror — same shape the button visibility uses, so the two can't diverge.
- Deleting `WeatherDisplay.todaySummary` alongside `legacyAlert` is the right cleanup: that field only existed to feed the alert title, and leaving it would have been a "someday" chore that never lands. `WeatherForecast.todaySummary` in OBAKitCore is untouched — good, that's the actual API contract.
- Commit graph is easy to follow: one commit per logical step (wrapper → presentation swap → deletion → test prune → animation fix), no `Co-Authored-By` trailers, single-line subjects.
